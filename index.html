End-to-End Architecture

From raw documents to searchable vector embeddings in five well-defined phases.

1

Extraction

Parse PDFs, DOCX, HTML with Unstructured.io. VLM fallback for complex tables.

2

Chunking

Header-aware splitter preserves document hierarchy. Configurable token limits.

3

Enrichment

LLM generates summaries. Title + context prepended to every chunk.

4

Indexing

BGE-m3 embeddings + bulk index into OpenSearch. k-NN + BM25 hybrid search.

Capabilities

Why SemantixRAG?

Every feature is designed to solve real-world RAG ingestion challenges.

Structure-Preserving Extraction

Documents are parsed with full structural fidelity. Headers, tables, lists, and figures are extracted with their relationships intact — not just flat text.

Unstructured.ioPyMuPDFElement Mapping

Smart Table Fallback

When standard OCR fails on complex tables, page images are routed to a Vision-Language Model (LLaVA / ColPali) that returns perfect Markdown tables.

VLMOllamaMarkdown

Header-Aware Chunking

Paragraphs are grouped under their parent headers. The chunker respects document hierarchy, preventing orphaned sentences that lose semantic meaning.

Semantic BoundariesOverlapConfigurable

Contextual Enrichment

Every chunk gets the document title and an LLM-generated summary prepended. This prevents the "lost in the middle" problem during retrieval.

GemmaLlamaOllama

BGE-m3 Embeddings

State-of-the-art multi-lingual dense embeddings via HuggingFace sentence-transformers. Auto-detects CUDA, MPS, or CPU. Graceful mock fallback.

1024-dimMulti-lingualNormalized

Hybrid Search (k-NN + BM25)

Combines dense vector search with traditional keyword search via Reciprocal Rank Fusion. Catches both semantic similarity and exact keyword matches.

HNSWCosine SimilarityRRF

CDC & Incremental Updates

Python Watchdog monitors directories for file changes. When a document is updated, old chunks are deleted and the document is re-indexed automatically.

WatchdogAuto-syncDelete-by-ID

100% Local & Open Source

Zero cloud dependencies. Everything runs on your hardware with Docker. Embedding models and LLMs run locally via Ollama or HuggingFace.

Quick Start

From Zero to Search in 5 Commands

Get the pipeline running on your machine in minutes.

SemantixRAG
# 1. Clone the repo
$ git clone https://github.com/SemantixRAG/SemantixRAG.git
$ cd SemantixRAG

# 2. Start OpenSearch (k-NN enabled)
$ cd docker && docker-compose up -d
✔ Container rag-opensearch Started
✔ Container rag-opensearch-dashboards Started

# 3. Install Python dependencies
$ pip install -r requirements.txt
✔ All 24 packages installed

# 4. Create the index and ingest a document
$ python main.py init
✔ Index 'rag_documents' created (knn_vector, 1024 dim)
$ python main.py ingest ./documents/sample_document.md
✔ Extracted 12 elements → 5 chunks → 5 embeddings indexed

# 5. Hybrid search!
$ python main.py search "reinforcement learning agent"
